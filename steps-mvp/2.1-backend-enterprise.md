## PROMPT DÀNH CHO AI AGENT: XÂY DỰNG BACKEND ENTERPRISE ARCHITECTURE

```prompt
Bạn là một Senior Backend Engineer. Hãy xây dựng toàn bộ mã nguồn Express.js + TypeScript cho dự án An Nhiên trong thư mục `backend/` theo kiến trúc Layered Architecture chuẩn Enterprise để tối đa hóa điểm đánh giá từ công cụ tự động (Codex).

### 1. Cài đặt thêm thư viện bắt buộc
Trước khi viết code, hãy chắc chắn `backend/package.json` có các thư viện sau:
`zod` (Input Validation), `crypto-js` (Mã hóa E2EE thực sự), `bcryptjs` (Mã hóa password), `express-async-errors` (Xử lý lỗi async).
Thêm vào dependencies nếu chưa có. Cài thêm `@types/bcryptjs`.

### 2. Kiến trúc Thư mục (Layered Architecture)
Tạo cấu trúc chia tách rõ ràng (Design Pattern):
- `src/controllers/`: Chứa logic xử lý HTTP request/response.
- `src/services/`: Chứa core business logic (gọi Supabase DB, gọi Gemini AI).
- `src/routes/`: Định nghĩa API endpoints và gắn middleware/controller.
- `src/middlewares/`: JWT Auth, Zod Validation, Global Error Handler.
- `src/validations/`: Chứa các Zod schemas.
- `src/utils/`: Helper functions (Crypto AES-256, Logger).

### 3. Triển khai các tính năng cốt lõi:

#### A. Input Validation (Zod) & Error Handling
- Viết `src/middlewares/validate.ts` nhận vào Zod schema để kiểm tra `req.body`, `req.query`.
- Viết `src/middlewares/errorHandler.ts`: Bắt toàn bộ lỗi từ Controller. Trả về JSON chuẩn: `{ success: false, error: { code, message, details } }`. Phân biệt lỗi Zod (400) và lỗi server (500).

#### B. Bảo mật (Cryptography & Auth)
- Viết `src/utils/crypto.ts` dùng `crypto-js` để định nghĩa hàm `encryptAES` và `decryptAES`.
- Logic Auth: `POST /api/auth/setup` chỉ tạo User ẩn danh. `POST /api/auth/login` nhận `{ username, password }`, sử dụng `bcryptjs.compare` để đối chiếu với `password_hash` trong DB. Trả về JWT.

#### C. Database Layer (Supabase Service)
- Viết `src/lib/supabase.ts` khởi tạo client bypass RLS.
- Viết các service như `JournalService`, `PostService` tương tác với DB qua type-safe queries.

#### D. AI Integration Layer (Gemini Service)
- Viết `src/services/ai.service.ts` gọi `@google/generative-ai`.
- BẮT BUỘC: Prompt của hàm `triage` (phân tích nguy cơ) và `moderation` (kiểm duyệt) phải yêu cầu trả về JSON chuẩn, có error handling nếu LLM trả về format sai (fallback về riskLevel = 'medium' an toàn).
- Quản lý 10 Persona Prompts cho AI Chatbot (Linh, Minh, Hà An, Bác sĩ Hương, v.v...) bằng một dictionary/map có type rõ ràng.

### 4. Triển khai 19 API Endpoints (tương tự API Contract):
- `Auth`: POST `/api/auth/setup`, POST `/api/auth/login`, GET `/api/auth/me`, PATCH `/api/auth/status`.
- `Journals`: POST `/api/journals`, GET `/api/journals/me`, GET `/api/journals/:id`.
- `Posts`: GET `/api/posts`, POST `/api/posts`, POST `/api/posts/:id/react`, GET `/api/posts/flagged`, PATCH `/api/posts/:id/status`.
- `Conversations`: POST `/api/conversations`, GET `/api/conversations/queue`, PATCH `/api/conversations/:id/accept`, PATCH `/api/conversations/:id/close`, POST `/api/conversations/:id/transfer`.
- `Messages`: POST `/api/messages`, GET `/api/messages/:conversationId`.
- `Videos`: GET `/api/videos`, POST `/api/videos`.

### 5. Yêu cầu Code Style
- Mọi hàm đều phải có JSDoc comments bằng tiếng Anh mô tả param, return.
- KHÔNG dùng `any`. Định nghĩa Interface/Type TypeScript cho toàn bộ DTOs và Database rows.
- File `src/index.ts` kết nối tất cả các route và inject Global Error Handler ở cuối cùng.

Hãy tạo ra tất cả các thư mục, file cấu hình và nội dung code theo đúng chỉ dẫn Enterprise trên.
```

## KIỂM TRA & XÁC MINH (Checklist cuối bước 2.1)

Sau khi AI chạy xong prompt này, hãy kiểm tra:

- [ ] Trong `backend/src/` có đủ các thư mục `controllers/`, `services/`, `middlewares/`, `routes/`, `validations/`.
- [ ] Mở file `package.json` kiểm tra đã có `zod` và `crypto-js`.
- [ ] Gửi request sai (thiếu trường bắt buộc) tới API tạo bài đăng để xem middleware Zod có trả về lỗi 400 chuẩn JSON không.
- [ ] Chạy `npm run build` không có lỗi TypeScript nào (`no any`).
